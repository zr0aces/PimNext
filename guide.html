02 CUPS Print Server Canon Driver · AirPrint · IPP
2.1 Choose a Canon Driver — CAPT vs UFR II LT ›

The LBP7110cw supports two Canon Linux driver stacks. Choose one based on your needs, then use the matching Dockerfile below.

Driver Type Print Quality Architecture Best For
Canon CAPT
cndrvcups-capt		 Proprietary CAPT protocol Good — standard output i386 (32-bit .deb) Simple setup, legacy systems
Canon UFR II LT
cndrvcups-ufr2lt		 UFR II — Canon's modern raster Better — richer color fidelity amd64 + arm64 native Raspberry Pi 4, better quality
💡
Recommendation: Use UFR II LT on Raspberry Pi 4 (arm64) — it has native arm64 support and better color output. Use CAPT on older x86 hosts or if UFR II has issues with your specific firmware version.
cups/Dockerfile — CAPT
FROM ubuntu:20.04
ENV DEBIAN_FRONTEND=noninteractive

RUN apt-get update && apt-get install -y \
    cups cups-client cups-filters cups-bsd \
    avahi-daemon avahi-utils libnss-mdns \
    wget libcups2 libcupsimage2 \
    && rm -rf /var/lib/apt/lists/*

# Canon CAPT driver — LBP7110cw, 32-bit .deb
# Requires i386 architecture on amd64 host
RUN dpkg --add-architecture i386 \
  && apt-get update \
  && apt-get install -y libc6:i386 libstdc++6:i386

RUN wget -q -O /tmp/capt-common.deb \
    "https://gdlp01.c-wss.com/gds/8/0100004508/11/cndrvcups-common-3.21-1.i386.deb" \
  && wget -q -O /tmp/capt.deb \
    "https://gdlp01.c-wss.com/gds/8/0100004508/11/cndrvcups-capt-2.71-1.i386.deb" \
  && dpkg -i /tmp/capt-common.deb /tmp/capt.deb || apt-get install -f -y \
  && rm /tmp/capt-common.deb /tmp/capt.deb

# Open CUPS to LAN, enable printer sharing
RUN sed -i 's/Listen localhost:631/Listen 0.0.0.0:631/' /etc/cups/cupsd.conf \
  && cupsctl --share-printers --remote-any

EXPOSE 631
CMD service avahi-daemon start && /usr/sbin/cupsd -f
⚠️
CAPT note: The CAPT driver requires a separate daemon (ccpd) to be running alongside CUPS. After adding the printer in CUPS Web UI, also run: docker exec cups-server /etc/init.d/ccpd start
cups/Dockerfile — UFR II LT
FROM ubuntu:22.04
ENV DEBIAN_FRONTEND=noninteractive

RUN apt-get update && apt-get install -y \
    cups cups-client cups-filters cups-bsd \
    avahi-daemon avahi-utils libnss-mdns \
    wget libcups2 libcupsimage2 \
    libxml2 libpng16-16 \
    && rm -rf /var/lib/apt/lists/*

# Canon UFR II LT driver — native amd64 + arm64
# Supports LBP7110cw with better color rendering
RUN wget -q -O /tmp/ufr2lt-common.deb \
    "https://gdlp01.c-wss.com/gds/1/0100010108/03/cnrdrvcups-ufr2lt-uk-3.90-1.04.x86_64.deb" \
  && dpkg -i /tmp/ufr2lt-common.deb || apt-get install -f -y \
  && rm /tmp/ufr2lt-common.deb

# Register UFR II LT PPD with CUPS
RUN /usr/sbin/canonLBP setup 2>/dev/null || true

# Open CUPS to LAN, enable printer sharing
RUN sed -i 's/Listen localhost:631/Listen 0.0.0.0:631/' /etc/cups/cupsd.conf \
  && cupsctl --share-printers --remote-any

EXPOSE 631
CMD service avahi-daemon start && /usr/sbin/cupsd -f
✅
UFR II LT note: No separate daemon needed — the driver integrates directly as a CUPS filter. Works cleanly on arm64 (Raspberry Pi 4). When adding the printer in CUPS, choose driver: Canon LBP7110C UFR II from the list.
After choosing either driver — add printer in CUPS Web UI
  1. Open http://[host-ip]:631 → Administration → Add Printer
  2. Choose AppSocket/HP JetDirect → URI: socket://192.168.1.50:9100
  3. CAPT: select Canon LBP7110C CAPT from driver list
  4. UFR II LT: select Canon LBP7110C UFR II from driver list
  5. Enable "Share This Printer" → Set as Default → Save
2.2 Create master docker-compose.yml ›
docker-compose.yml
services:

  # ── CUPS: AirPrint + IPP ─────────────────────────────
  cups:
    build: ./cups
    container_name: cups-server
    restart: unless-stopped
    network_mode: host          # required for mDNS / AirPrint
    volumes:
      - ./cups/config:/etc/cups
      - ./cups/logs:/var/log/cups
      - /var/run/cups:/var/run/cups   # share socket with printbot

  # ── PrintBot v1.0.0 — pre-built GHCR image ──────────
  printbot:
    image: ghcr.io/zr0aces/printbot:1.0.0   # no build needed!
    container_name: printbot
    restart: always
    env_file: .env
    volumes:
      - /var/run/cups/cups.sock:/var/run/cups/cups.sock
    depends_on:
      - cups
    # LBP7110cw is Wi-Fi — remove /dev/usb/lp0 device line

  # ── Home Assistant ───────────────────────────────────
  homeassistant:
    image: ghcr.io/home-assistant/home-assistant:stable
    container_name: homeassistant
    restart: unless-stopped
    network_mode: host
    volumes:
      - ./ha/config:/config
      - /var/run/docker.sock:/var/run/docker.sock  # needed for command_line sensors
✅
No build step! ghcr.io/zr0aces/printbot:1.0.0 is pre-built for both amd64 and arm64. Docker pulls it automatically.

3.4 Print Options — send before the file ›

Send a text message with options before sending the file. Options apply to the next print only, then reset to defaults (color, 1 copy).

Message Effect
bw or gray Print in black & white (-o ColorModel=Gray)
2x / 3x / 4x Print 2, 3, or 4 copies (-n N)
bw 2x Combine — B&W + 2 copies
anything else Bot replies: "Unknown option. Use: bw, 2x, 3x, bw 2x"
💡
Bot confirms your options: "⚙️ Next print: B&W, 2 copies" — then send the file.

4.4 CUPS Sensors — printer state, job count, page count ›

Add all three sensors to configuration.yaml. They poll CUPS every 30–60 seconds via lpstat and lpinfo inside the cups container.

ha/config/configuration.yaml
command_line:

  # ── Sensor 1: Printer State ────────────────────────────
  # Values: idle · printing · stopped · offline
  - sensor:
      name: "Printer State"
      unique_id: cups_printer_state
      command: >
        docker exec cups-server lpstat -p Canon-LBP7110cw 2>&1 || echo "offline"
      scan_interval: 30
      value_template: >
        {% set v = value | lower %}
        {% if 'idle' in v %}idle
        {% elif 'printing' in v %}printing
        {% elif 'stopped' in v %}stopped
        {% else %}offline{% endif %}
      icon: >
        {% if value == 'printing' %}mdi:printer-pos-active
        {% elif value == 'offline' %}mdi:printer-off
        {% elif value == 'stopped' %}mdi:printer-alert
        {% else %}mdi:printer-check{% endif %}

  # ── Sensor 2: Active Job Count ─────────────────────────
  # Counts lines in lpstat -o output (each line = one job)
  - sensor:
      name: "Print Jobs Queued"
      unique_id: cups_jobs_queued
      command: >
        docker exec cups-server sh -c
        "lpstat -o 2>/dev/null | grep -c '' || echo 0"
      scan_interval: 30
      value_template: "{{ value | trim | int(0) }}"
      unit_of_measurement: "jobs"
      icon: mdi:file-multiple

  # ── Sensor 3: Total Pages Printed ──────────────────────
  # Reads CUPS page_log — counts completed pages historically
  - sensor:
      name: "Total Pages Printed"
      unique_id: cups_pages_total
      command: >
        docker exec cups-server sh -c
        "wc -l < /var/log/cups/page_log 2>/dev/null || echo 0"
      scan_interval: 300
      value_template: "{{ value | trim | int(0) }}"
      unit_of_measurement: "pages"
      icon: mdi:counter

  # ── Sensor 4: CUPS Server Reachable ────────────────────
  - binary_sensor:
      name: "CUPS Online"
      unique_id: cups_online
      command: >
        docker exec cups-server lpstat -r 2>&1
      scan_interval: 30
      payload_on: "scheduler is running"
      payload_off: "offline"
      device_class: connectivity
💡
Printer name: Replace Canon-LBP7110cw with the exact name you set in CUPS Web UI. Verify with: docker exec cups-server lpstat -p
4.5 CUPS REST API sensor — richer printer attributes ›

CUPS exposes a REST-like JSON endpoint. Use HA's rest sensor to pull printer attributes including state reason and accepting status directly from the CUPS API.

ha/config/configuration.yaml (append)
rest:
  # Polls the CUPS IPP endpoint for rich printer attributes
  - resource: "http://[host-ip]:631/printers/Canon-LBP7110cw"
    scan_interval: 60
    headers:
      Accept: application/json
    sensor:
      - name: "Printer State Reason"
        unique_id: cups_state_reason
        value_template: >
          {{ value_json.get('printer-state-message', 'none') | default('none') }}
        icon: mdi:information-outline

      - name: "Printer Accepting Jobs"
        unique_id: cups_accepting
        value_template: >
          {{ value_json.get('printer-is-accepting-jobs', false) }}
        icon: mdi:checkbox-marked-circle
⚠️
The CUPS REST JSON response format varies by version. If this returns errors, use only the command_line sensors from step 4.4 — they are more reliable.
4.6 Template sensors — derived state for dashboard ›

Add helper template sensors for cleaner dashboard display and automation conditions:

ha/config/configuration.yaml (append)
template:
  - sensor:
      # Human-readable printer status with emoji
      - name: "Printer Status Display"
        unique_id: printer_status_display
        state: >
          {% set s = states('sensor.printer_state') %}
          {% if s == 'idle' %}✅ Ready
          {% elif s == 'printing' %}🖨️ Printing...
          {% elif s == 'stopped' %}⚠️ Stopped
          {% elif s == 'offline' %}❌ Offline
          {% else %}❓ Unknown{% endif %}
        icon: >
          {% set s = states('sensor.printer_state') %}
          {% if s == 'printing' %}mdi:printer-pos-active
          {% elif s == 'offline' or s == 'stopped' %}mdi:printer-alert
          {% else %}mdi:printer{% endif %}

      # Today's print job count (resets at midnight)
      - name: "Jobs Today"
        unique_id: cups_jobs_today
        state: >
          {% set total = states('sensor.total_pages_printed') | int(0) %}
          {{ total }}
        unit_of_measurement: "pages"

  - binary_sensor:
      # True when printer needs attention
      - name: "Printer Needs Attention"
        unique_id: printer_needs_attention
        state: >
          {{ states('sensor.printer_state') in ['offline', 'stopped'] }}
        device_class: problem
4.7 Automations — full set ›
ha/config/automations.yaml
# ── 1. PrintBot job webhook → HA notification ──────────
- alias: "Print — Job Received"
  id: print_job_received
  trigger:
    - platform: event
      event_type: printbot_job_sent
  action:
    - service: notify.persistent_notification
      data:
        title: "🖨️ Print Job Sent"
        message: >
          File: {{ trigger.event.data.file_name }}
          Mode: {{ 'B&W' if not trigger.event.data.color else 'Color' }}
          Copies: {{ trigger.event.data.copies }}
    - service: notify.telegram
      data:
        message: >
          🖨️ Printing {{ trigger.event.data.file_name }}
          ({{ 'B&W' if not trigger.event.data.color else 'Color' }},
          {{ trigger.event.data.copies }}x)

# ── 2. Printer offline alert (debounced 2 min) ──────────
- alias: "Print — Printer Offline Alert"
  id: printer_offline_alert
  trigger:
    - platform: state
      entity_id: sensor.printer_state
      to: "offline"
      for: "00:02:00"
  action:
    - service: notify.telegram
      data:
        message: >
          ⚠️ Canon printer is offline!
          Check CUPS: http://[host-ip]:631

# ── 3. Printer back online ──────────────────────────────
- alias: "Print — Printer Back Online"
  id: printer_back_online
  trigger:
    - platform: state
      entity_id: sensor.printer_state
      from: "offline"
      to: "idle"
  action:
    - service: notify.telegram
      data:
        message: "✅ Printer is back online and ready."

# ── 4. Printer stopped / error ──────────────────────────
- alias: "Print — Printer Stopped"
  id: printer_stopped
  trigger:
    - platform: state
      entity_id: sensor.printer_state
      to: "stopped"
  action:
    - service: notify.telegram
      data:
        message: >
          🛑 Printer stopped — may need attention.
          Check CUPS: http://[host-ip]:631

# ── 5. Daily print summary at 9pm ───────────────────────
- alias: "Print — Daily Summary"
  id: print_daily_summary
  trigger:
    - platform: time
      at: "21:00:00"
  condition:
    - condition: template
      value_template: >
        {{ states('sensor.print_jobs_queued') | int(0) == 0 }}
  action:
    - service: notify.telegram
      data:
        message: >
          📊 Daily Print Summary
          Printer: {{ states('sensor.printer_status_display') }}
          Total pages logged: {{ states('sensor.total_pages_printed') }}

# ── 6. Alert if print queue stuck > 10 min ──────────────
- alias: "Print — Queue Stuck"
  id: print_queue_stuck
  trigger:
    - platform: numeric_state
      entity_id: sensor.print_jobs_queued
      above: 0
      for: "00:10:00"
  action:
    - service: notify.telegram
      data:
        message: >
          ⏳ Print queue has {{ states('sensor.print_jobs_queued') }} job(s)
          stuck for over 10 minutes. Check CUPS: http://[host-ip]:631

06 Troubleshooting Common Issues
Bot says "lp not found"
The GHCR image includes cups-bsd. If using a custom build, add it to the Dockerfile. Verify: docker exec printbot which lp
printbot can't reach CUPS socket
Check ls /var/run/cups/cups.sock exists on host. CUPS must start before printbot — use depends_on: cups in compose.
AirPrint invisible on iPhone
CUPS must use network_mode: host. mDNS multicast won't cross Docker's bridge network.
Windows IPP "can't connect"
Open firewall: sudo ufw allow 631. Confirm CUPS is running: docker ps | grep cups.
/status shows no printers
Re-add printer at :631/admin. Check ./cups/config/printers.conf is persisted in the volume.
HA webhook not firing
Ensure both HA_URL and HA_TOKEN are set and uncommented in .env. Restart printbot after editing.
Print queued but nothing prints
Test: docker exec cups-server ping 192.168.1.50. Check printer isn't paused in CUPS Web UI under Printers.
HA sensors show "unknown"
Mount the Docker socket into HA so it can run docker exec. Add - /var/run/docker.sock:/var/run/docker.sock to the homeassistant volumes in compose, then restart.
sensor.total_pages_printed always 0
Enable page logging in CUPS: docker exec cups-server sh -c "echo 'PageLog /var/log/cups/page_log' >> /etc/cups/cupsd.conf" then restart CUPS.
arm64 / Raspberry Pi image error
Use ghcr.io/zr0aces/printbot:1.0.0 — it's multi-arch. Don't build locally unless needed; the GHCR image is pre-built for arm64.
